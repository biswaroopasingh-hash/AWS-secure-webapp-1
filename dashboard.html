Services Explained
🐙
GitHub Repository
SOURCE CONTROL
Stores your index.html frontend code. Every push to main branch automatically triggers a redeployment on Amplify. This is your CI/CD pipeline.
Version Control Auto Deploy CI/CD
🌐
AWS Amplify
FRONTEND HOSTING
Hosts your HTML/CSS/JS website with a live HTTPS URL. Connects directly to GitHub and redeploys automatically. Provides free SSL certificate and a global CDN for fast loading worldwide.
HTTPS CDN Free SSL Auto Redeploy
🔐
Amazon Cognito
AUTHENTICATION
Manages user login and registration with email/password. Issues JWT tokens after login. API Gateway checks these tokens — only verified users can submit the form and call the backend API.
JWT Tokens User Pool Email Auth
🔀
API Gateway
HTTP API — POST /submit
Acts as the front door for your backend. Receives the form POST request from the browser, verifies the Cognito JWT token, then forwards the request to Lambda. Also handles CORS headers.
REST API CORS JWT Authorizer
⚡
AWS Lambda
SERVERLESS BACKEND · PYTHON
Runs your Python backend logic without any server. Receives the form data (name + email), parses it, and saves it to DynamoDB. Scales automatically — runs only when called, so you pay nothing when idle.
Python 3.12 No Server boto3 Auto Scale
🗄️
Amazon DynamoDB
NOSQL DATABASE
Permanently stores all form submissions. Table name: ContactForm. Uses email as the primary key. Serverless — no database server to manage. Scales to millions of records automatically.
NoSQL Serverless Primary Key: email
🗝️
IAM Role
PERMISSIONS & ACCESS
Grants Lambda permission to write to DynamoDB. Policy used: AmazonDynamoDBFullAccess. Without this role, Lambda would be blocked from saving data — security principle of least privilege.
Permissions IAM Policy Least Privilege
📊
Amazon CloudWatch
MONITORING & LOGS
Automatically captures all Lambda logs including your print() statements. Dashboard shows invocations, errors, and duration. Alarms alert you when errors exceed threshold.
Logs Dashboard Alarms Metrics
🛡️
AWS WAF
WEB APPLICATION FIREWALL
Sits in front of API Gateway and blocks malicious traffic before it reaches your backend. Protects against SQL Injection, XSS attacks, and known bad IPs. Uses AWS Managed Core Rule Set — no manual rule writing needed.
SQL Injection XSS Block Core Rule Set IP Reputation
Request Journey — Step by Step
1
User opens website Browser loads index.html from AWS Amplify over HTTPS. The page shows the contact form with Name and Email fields.
2
WAF scans the traffic All requests first pass through AWS WAF. Malicious payloads (SQL injection, XSS scripts) are blocked immediately before reaching any service.
3
User logs in via Cognito Cognito verifies email + password and issues a JWT token. This token proves the user is authenticated.
4
Form submitted → API Gateway JavaScript sends a POST request to API Gateway endpoint /submit with the JWT token in the Authorization header and form data in the body (JSON).
5
API Gateway validates JWT The Cognito Authorizer checks the JWT token. If invalid or expired, request is rejected with 401 Unauthorized. Valid requests proceed to Lambda.
6
Lambda executes Python code Lambda receives the event, parses the JSON body (name + email), logs the data to CloudWatch, then calls DynamoDB to save the record.
7
DynamoDB stores the data The ContactForm table permanently saves the name and email using email as the primary key. Returns success to Lambda.
8
Response returns to user Lambda returns statusCode 200 → API Gateway forwards it → Browser shows ✅ "Submitted successfully!" message to the user.
9
CloudWatch logs everything All Lambda print() logs, execution time, memory usage, and any errors are automatically captured and visible in CloudWatch Log Groups.
